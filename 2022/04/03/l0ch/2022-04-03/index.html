[하루한줄] CVE-2022-22965: Spring 프레임워크의 Spring4Shell 취약점

URL

Spring4Shell: Security Analysis of the latest Java RCE ‘0-day’ vulnerabilities in Spring

Target

  • Containers

Explain

Java 플랫폼의 오픈소스 프레임워크인 Spring Framework에서 원격 코드 실행이 가능한 취약점의 세부 정보가 공개되었습니다. Spring은 다양한 서비스에서 사용되는 인기있는 프레임워크로, 해당 취약점의 파급력이 클 것으로 예상되며 Log4j의 RCE 취약점이었던 Log4Shell과 유사한 Spring4Shell, SpringShell 등으로 불리고 있습니다.

2월에는 Spring Framework 관련 라이브러리에서 SpEL(Spring Expression Language) 를 악용한 취약점이 공개되기도 했습니다.

취약점은 Spring Core의 바인딩 프로세스 bindRequestParameters가 캐시의 객체 속성을 가져오기 위해 getCachedIntrospectionResults 메서드를 호출할 때 발생합니다.

getCachedIntrospectionResults 메서드는 위 사진과 같이 반환값에 클래스 객체가 포함되어 있습니다.
